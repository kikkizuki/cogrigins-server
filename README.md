# Minecraft Server "Cogrigins" - Fabric Edition

Этот репозиторий содержит конфигурационные файлы для сервера Minecraft на базе Fabric с установленными модификациями. Сервер развернут с использованием Docker Compose, что обеспечивает удобство и воспроизводимость.

## Описание

Сервер настроен для режима выживания (survival) с включенным PvP и генерацией структур. Используется seed `-5175376072305823271`.  Установлены многочисленные модификации, расширяющие функциональность игры.

* **FRP Port:** `59378`

### Безопасность и уровни доступа

Система использует гибридную модель доступа для удобства управления файлами без ущерба для безопасности хоста:

* **Уровень Администратора (Root)**: Полный доступ к системе и управлению Docker. Вход разрешен исключительно по **SSH-ключам** (ED25519). Парольный вход для `root` отключен.
* **Уровень Сервиса (SFTP)**: Управление файлами игрового сервера осуществляется через **сервисный аккаунт**. 
    * Для этого аккаунта разрешен вход по паролю, но он жестко ограничен протоколом `internal-sftp`. 
    * Прямой доступ к командной оболочке (Bash/SSH) для этого пользователя закрыт.
* **Изоляция**: Игровой сервер запущен в Docker-контейнере от имени сервисного аккаунта, что обеспечивает корректные права на файлы при работе через SFTP и предотвращает конфликты прав (Permission Issues).

## Запуск сервера с помощью Docker Compose

1.  Убедитесь, что у вас установлен Docker и Docker Compose.
2.  Перейдите в каталог, содержащий файлы `docker-compose.yml`, `fabric-server-launcher.properties` и папку `mods`.
3.  Запустите сервер командой:

    ```bash
    docker compose up -d
    ```

    `-d` означает запуск сервера в фоновом режиме (detached).

4.  Для остановки сервера используйте команду:

    ```bash
    docker compose down
    ```

## скрипты

В репозитории находятся инструменты для управления инфраструктурой и кастомизации геймплея:

### Системные скрипты (Root)
* **`backup.sh`**: Создает `.tar.gz` архив. Настроен через `cron` на ежедневное выполнение в 04:00.
* **`start.sh`**: Скрипт запуска с оптимизированными флагами **G1GC (Aikar's Flags)**.
    * **Память:** 10 ГБ выделено под JVM, 12 ГБ — лимит контейнера (запас для системы).
    * **CPU:** Оптимизация под 8 потоков i5-12400 для минимизации задержек (MaxGCPauseMillis=50).

### Игровая логика (KubeJS)
Путь: `./kubejs/server_scripts/originscraft.js`

Для баланса экономики сервера добавлен кастомный рецепт **Orb of Origin** (Сфера Происхождения). В оригинальном моде предмет нельзя скрафтить, что ограничивало развитие. 

**Рецепт в `originscraft.js`:**
* **Сетка:** 8 × Алмаз + 1 × Звезда Незера (в центре).
* **Цель:** Сделать смену расы ценным эндгейм-событием, стимулируя фарм боссов.

## архивные скрипты

В папке archive находятся старые инструменты автоматизации. Это история перехода от ручного управления через screen к текущей архитектуре на Docker.

[ 1. backup.sh ]
      |
      +-- Механика: Симуляция действий игрока (say/save-all) в screen.
      +-- Проблема: Копирование мира "на лету" (cp) — риск битых чанков.
      +-- Статус: ЗАМЕНЕНО на Graceful Shutdown в Docker.

[ 2. restart.sh ]
      |
      +-- Механика: Бесконечный цикл мониторинга кода выхода Java.
      +-- Проблема: Отсутствие изоляции процессов и лимитов памяти.
      +-- Статус: ЗАМЕНЕНО на политику restart:always в Compose.

[ 3. restart_daily.sh ]
      |
      +-- Механика: Плановый рестарт с таймерами уведомлений игроков.
      +-- Проблема: Принудительный 'screen -X quit' при зависании Java.
      +-- Статус: ЗАМЕНЕНО на системные сигналы SIGTERM.

## 🛠 Техническая поддержка и история

Проект прошел путь от простых Bash-скриптов до масштабируемой Docker-инфраструктуры.

* **Текущий статус**: Стабильная работа на базе Docker + FRP.
* **Оптимизация**: Применяются флаги Aikar's (G1GC) для Java 21.
* **Логирование**: Настроена ротация логов и автоматические бэкапы через Graceful Shutdown.
